# 프롬프트 엔지니어링 전략

## 1. 프로젝트 개요

본 프로젝트는 임대차 계약 관련 법령, 판례, 법령해석례를 기반으로  
사용자 질문에 답변하는 RAG 기반 LLM 챗봇 시스템입니다.

대상 사용자: 부동산 계약 경험이 부족한 사회초년생 및 임차인

---

## 2. 시스템 프롬프트 (System Prompt)

LLM에게 부여할 역할과 제약 조건을 정의합니다.
```python
SYSTEM_PROMPT = """
당신은 임대차 계약 전문 AI 어시스턴트입니다.
주택임대차보호법, 관련 판례, 법령해석례를 기반으로 사용자의 질문에 답변합니다.

[역할]
- 임차인의 권리와 의무를 정확하게 안내합니다.
- 복잡한 법률 용어를 일반인도 이해할 수 있도록 쉽게 설명합니다.
- 제공된 법령/판례/해석례 문서만을 근거로 답변합니다.

[제약 조건]
- 제공된 문서에 없는 내용은 "해당 내용은 제공된 문서에서 확인할 수 없습니다"라고 답변합니다.
- 법률적 판단이나 개인 법률 자문은 제공하지 않습니다.
- 반드시 근거 문서(법령명, 판례번호 등)를 함께 제시합니다.
- 할루시네이션 방지를 위해 문서에 명시된 내용만 인용합니다.
"""
```

---

## 3. RAG 문서 삽입 템플릿

검색된 법령/판례/해석례를 LLM에게 전달하는 형식입니다.
```python
RAG_TEMPLATE = """
[참고 문서]
{context}

[사용자 질문]
{question}

[답변 지침]
- 위 참고 문서를 근거로 답변하세요.
- 답변 마지막에 참고한 문서 출처를 명시하세요.
- 문서에 없는 내용은 추측하지 마세요.
"""
```

### 3-1. context 구성 방식

RAG로 검색된 문서를 아래 형식으로 정리하여 context에 삽입합니다.
```
[법령] 주택임대차보호법 제6조
내용: ...

[판례] 대법원 2023다12345
내용: ...

[해석례] 법령해석례 제2023-123호
내용: ...
```

## 4. 사용자 입력 유형별 처리 방식

회의에서 논의된 사용자 입력 시나리오 3가지에 대한 처리 방식입니다.

### 유형 1: 특약 직접 입력
> "반려동물을 키우고 있는데 문제가 없는 특약을 추가하고 싶어"

- 사용자가 원하는 특약 내용을 직접 입력
- 해당 특약의 법적 유효성 및 관련 법령 검색 후 안내

### 유형 2: 상황 설명형 입력
> "1년 계약을 했는데 중간에 방을 빼야 하는 상황이야"

- 사용자 상황을 분석하여 관련 법령/판례 검색
- 임차인 권리(계약해지, 보증금 반환 등) 중심으로 안내

### 유형 3: 계약서 특약 검토
> "계약서에 이런 특약이 있는데 문제없어?"

- 입력된 특약 내용을 법령/판례 기준으로 검토
- 불리한 조항 여부 및 수정 방향 안내

## 5. 출력 형식 (Answer Format)
```python
ANSWER_FORMAT = """
**답변 형식**

1. 핵심 답변 (2~3문장 요약)
2. 상세 설명 (관련 법령/판례 근거 포함)
3. 참고 문서
   - 법령: {법령명} {조항}
   - 판례: {사건번호} ({선고일자})
   - 해석례: {해석례번호}
4. 주의사항 (필요 시)
   - 본 답변은 법률 자문이 아니며, 구체적인 사안은 법률 전문가와 상담하세요.
"""
```

## 6. 할루시네이션 방지 전략

| 전략 | 설명 |
|---|---|
| 문서 기반 답변 강제 | 시스템 프롬프트에서 제공된 문서 외 내용 답변 금지 명시 |
| 출처 명시 의무화 | 모든 답변에 법령명/판례번호 등 출처 포함 |
| 불확실 시 명시 | 문서에 없는 내용은 "확인 불가" 명시 |
| top-k 문서 제한 | RAG 검색 결과 상위 3~5개만 context로 사용 |

---

## 7. 추후 개선 방향

- 사용자 입력 의도 분류기 추가 (특약 검토 / 권리 안내 / 분쟁 해결)
- 멀티턴(Multi-turn Conversation) 대화 지원 (이전 대화 내용 기억하면서 이어지는 대화)
- Tavily 웹 검색 연동으로 최신 판례 보완
